Robothon Rules

▸ Theme

Build a MuJoCo-based robot simulator, task scenario or data-collection system; submit a runnable simulation task, interactive system or data-collection environment.

▸ Embodiment

Any embodiment: robotic arms, mobile robots, quadrupeds, humanoids, grippers, autonomous vehicles, dexterous / multi-finger hands, and more.

▸ Eligibility

Any agent is allowed (Claude Code / Codex / Gemini CLI / Cursor / Kimi / your own, etc.); note the tools used on submission.

▸ Tech Stack

MuJoCo as the primary physics engine; make full use of MJCF, collisions, joints, sensors and actuators (one-line pip install, all platforms, no GPU).

▸ Deliverables

Code repo + run instructions (dependencies / install / launch / controls) + MuJoCo scene & robot model + demo video (1–3 min, produced by running the submitted code) + project writeup.

▸ Results

A project's latest score counts (not its all-time best); anonymous submission allowed.

scoring.cfg×
Scoring Rubric
01Runnability — Whether the code runs smoothly and is easy to reproduce.
02MuJoCo Usage Depth — How thoroughly MJCF, physics simulation, collisions, joints, sensors and actuators are used.
03Task Design — Whether the task is clear, challenging and meaningful.
04Control — Teleoperation, autonomous control, policy control, task planning or data-collection capability.
05Dexterous Manipulation — If using a dexterous hand: multi-finger coordination, fine manipulation, high-DOF control.
06Engineering Quality — Code structure, docs, configuration and asset management clarity.
07Presentation — demo Whether the demo video is intuitive and convincing.
08Innovation — Novelty of the scenario, embodiment, task or application direction.

$ judges --anchor --median --avg

All three AI judges (Claude / GPT / Gemini) receive an identical review package (demo video + code + README / writeup). Anchored absolute scoring, each judge called 3 times and the median taken, then averaged across the three for the leaderboard. Fixed temperature, latest score only, daily rate limit — no gaming the system.
